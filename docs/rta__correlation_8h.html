void rta_correlation_fast ( rta_real_t *  correlation,
const unsigned int  c_size,
const rta_real_t *  input_vector_a,
const rta_real_t *  input_vector_b,
const unsigned int  filter_size 
)

Compute correlation between 'input_vector_a' and 'input_vector_b' into 'correlation'. If 'input_vector_a' == 'input_vector_b', it computes auto-correlation. This function can run in place if 'correlation' == 'input_vector_a' or 'correlation' == 'input_vector_b'.

\(C(i) = \sum_{f=0}^{filter\_size-1} A(f+i) \cdot B(f), i=\{0,c\_size-1\}\)

This function is unbiased by nature but misses some information, specially for the first coefficients. It should be negligible if 'filter_size' is much greater than 'c_size', like ('filter_size' / 'c_size' > 20).

Parameters
correlationsize is 'c_size'
c_sizeis the 'correlation' order + 1, 'c_size' must be > 0
input_vector_asize a_size must be >= 'c_size' + 'filter_size'
input_vector_bsize b_size must be >= 'c_size'
filter_sizeis the maximum shift for 'input_vector_a'. In practice, 'filter_size' == (a_size - 'c_size')
