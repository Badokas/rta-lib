Detailed Description

k-dimensional search tree

Author
Diemo Schwarz
Date
29.9.2008
Version
1.0

Efficient multidimensional binary search tree with logarithmic time complexity: From about 80-100 points, the number of comparisions doesn't rise any more perceptibly.

Dimensions can be weighted while building the tree, and while searching. The weight is 1 / sigma, just as in mnm.mahalanobis. Sigma = 0 means: ignore this dimension.

Call sequence for builing and using the tree:

  • 1. initialise tree structure with kdtree_init()
  • 2. set parameters like decomposition mode kdtree_t::dmode and mean mode kdtree_t::mmode, tree height adaptation kdtree_t::givenheight
  • 3. set data vector with kdtree_set_data(), this returns the number of nodes the tree will build
  • 4. initialise nodes, possibly providing memory allocated according the number of nodes returned above, with kdtree_init_nodes()
  • 5. optionally set weights for building with kdtree_set_sigma()
  • 6. build the tree with kdtree_build()
  • 7. then you can query the tree with kdtree_search_knn().

enum rta_kdtree_dmode_t

decomposition mode

Different ways to split data space at each tree level:

  • orthogonal to dimension (fastest tree-building speed),
  • by arbitrary hyperplane (for testing only), or
  • by PCA (optimal decomposition)
Enumerator
dmode_orthogonal 

optimised othogonal to axes

dmode_hyperplane 

hyperplane orthogonal to axes

dmode_pca 

hyperplane along principal components

initialise tree nodes

This function must be called after kdtree_set_data() and before kdtree_build(). It initialises the tree nodes and the vectors decomposing the search space.

Parameters
selfkd-tree structure
nodesspace for tree nodes or NULL for automatic allocation
meansspace for mean vectors or NULL for automatic allocation
planesspace for hyperplanes or NULL for automatic allocation

Prerequisites:

  • tree data(m, n) must have been set with rta_kdtree_set_data(), this returned nnodes
  • if nodes is not NULL, it must point to space for the tree nodes of size nnodes * sizeof(kdtree_node_t), which must have been allocated outside of the library
  • if means is not NULL, it must point to space for the mean vectors of size nnodes / 2 * ndim, which must have been allocated outside of the library
  • if planes is not NULL and decomposition mode is not dmode_orthogonal, it must point to space for split hyperplane base vectors of size nnodes / 2 * ndim, which must have been allocated outside of the library

Perform search in kd-tree structure t.

Return the k nearest neighbours in multi-dimensional data using the search tree t. The output vector d contains the squared distances to the r closest data vectors in kdtree_t::data according to the formula

\[ d_j = \sum_{j=0}^{ndim} \left( \frac{data_j - x_j}{\sigma} \right)^2 \]

Parameters
tkd-tree structure
xvector of kdtree_t::ndim elements to search nearest neighbours of
stridestride in vector x
kmax number of neighbours to find (actual number can be lower)
rmax squared distance of neighbours to find (r = 0 means no limit)
use_sigmause weights
youtput vector (size == r <= k) of (base, element) indices into original data kdtree_t::data
doutput vector (size == r <= k) of squared distances to data vectors
Returns
r = the number of actual neighbours found, 0 <= r <= k
